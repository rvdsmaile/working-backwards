# Working Backwards skills

When a user is shaping a new product, feature, or product bet, check `.agents/skills/` for a relevant Working Backwards skill before drafting an answer or implementation plan.

Use `start-working-backwards` for a new or unstructured initiative. Use a specialised skill directly only when the user asks for that stage. Keep initiative artifacts in `docs/working-backwards/<slug>/`; do not create them until the user agrees to persist the initiative.

Use `start-technical-delivery` only after an initiative has passed its handoff gate. It reads the completed product artifacts, discovers conventions in the chosen implementation repository, and writes technical artifacts in that repository. Do not assume a specific issue tracker or publish tracker items without explicit approval.

The framework is narrative-first. Do not present assumptions as evidence or invent customer research. Treat the internal FAQ as optional.
