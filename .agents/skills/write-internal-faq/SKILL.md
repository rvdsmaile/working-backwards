# Write Internal FAQ

Read `../../working-backwards/WORKFLOW.md` and use `../../working-backwards/templates/internal-faq.md`.

1. Ask only questions that affect feasibility, ownership, trade-offs, adoption, or success.
2. Give each answer a decision, assumption, or unresolved status.
3. Record resulting decisions and risks in `decision-log.md`.
4. Save to `internal-faq.md`; proceed to the challenge skills.
