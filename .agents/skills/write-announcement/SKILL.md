# Write Announcement

Read `../../working-backwards/WORKFLOW.md` and use `../../working-backwards/templates/announcement.md` as the structure.

1. Start with the customer's problem and setting, then introduce the change and the resulting experience.
2. Make concrete promises only when supported by a decision; mark uncertain details as assumptions or questions.
3. Avoid generic claims, implementation detail, Amazon framing, and invented launch dates.
4. Save to `announcement.md` and hand off to `write-faq`.
