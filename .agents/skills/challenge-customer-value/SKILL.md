# Challenge Customer Value

Read `../../working-backwards/WORKFLOW.md`. Review without rewriting the draft first.

1. Test whether the named customer would recognise the problem, care about the change, and understand the trade-off.
2. Challenge vague value claims, missing alternatives, adoption friction, and unclear boundaries.
3. Add findings to `critique.md` using the shared table; give every finding a severity and status of `open`.
4. Do not resolve findings. Hand off to `synthesize-working-backwards`.
