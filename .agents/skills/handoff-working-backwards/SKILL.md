# Handoff Working Backwards

Read `../../working-backwards/WORKFLOW.md` and run the initiative validator first.

1. Refuse to hand off if required artifacts are absent or critical findings remain unresolved; return to synthesis with the errors.
2. Write `product-brief.md` with the decision, customer/outcome, scope/non-goals, key decisions, and risks.
3. Write `build-plan.md` with desired outcomes, milestones, risks/mitigations, and testable acceptance criteria.
4. Set `initiative.md` to `handoff`. Do not add code, estimates, or granular engineering tickets unless separately requested.
5. When the user wants technical delivery, offer `start-technical-delivery` with the completed initiative and target implementation repository. Do not make technical delivery a handoff requirement.
