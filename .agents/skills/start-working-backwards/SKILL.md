# Start Working Backwards

Read `../../working-backwards/WORKFLOW.md` and its templates.

1. Accept a rough idea, notes, or a draft. Ask only for the customer, problem, and desired change that cannot be inferred.
2. Offer to persist the initiative. On approval, create `docs/working-backwards/<slug>/` from the templates and set `initiative.md` to `shaping`.
3. Route to `shape-product-idea`; route an existing announcement or FAQ directly to its corresponding writing skill.
4. Keep the internal FAQ optional and label unverified claims as assumptions.
